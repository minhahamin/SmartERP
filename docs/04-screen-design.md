# 4. 화면 설계

각 페이지는 **목적 / 주요 기능 / 화면 구조 / UI 요소 / 사용자 흐름** 순서로 기술한다. 라우팅 구조는 React Router 기준이며, 레이아웃은 좌측 다크 사이드바 + 우측 라이트 콘텐츠 영역의 공통 셸(Shell)을 공유한다.

## 4.0 공통 레이아웃 셸

- **목적**: 모든 인증된 페이지에서 일관된 탐색 경험 제공
- **구조**: `Sidebar(다크, 240px 고정)` + `Topbar(검색/알림/프로필)` + `Content(라이트, 가변폭, max-width 1440px)` + `AI 챗봇 플로팅 버튼(우하단 고정)`
- **사용자 흐름**: 로그인 성공 → 셸 마운트 → 역할에 따라 사이드바 메뉴 항목이 동적으로 필터링되어 렌더링

---

## 4.1 로그인 (`/login`)

- **목적**: 신원 확인 및 워크스페이스 진입
- **주요 기능**: 이메일/비밀번호 로그인, 에러 메시지(오류 횟수 경고), 비밀번호 찾기 링크
- **화면 구조**: 중앙 정렬 단일 카드(420px) — 좌측 50%는 브랜드 일러스트/카피, 우측 50%는 로그인 폼 (Desktop). 모바일에서는 폼만 전체 폭.
- **UI 요소**: 로고, Input(이메일), Input(비밀번호, 마스킹 토글), Checkbox(로그인 유지), Button(로그인, Primary), 에러 Alert
- **사용자 흐름**: 입력 → 클라이언트 유효성 검증 → API 호출 → 성공 시 Access/Refresh Token 저장(Zustand + httpOnly 쿠키) → `/dashboard` 리다이렉트. 5회 실패 시 잠금 카운트다운 표시.

## 4.2 대시보드 (`/dashboard`)

- **목적**: 로그인 직후 역할별 핵심 지표를 한눈에 파악
- **주요 기능**: KPI 카드, 추이 차트, 경고 위젯(재고/생산지연), 최근 공지, AI 챗봇 진입
- **화면 구조**: 상단 KPI 카드 4분할 grid → 중단 좌(매출/근태 추이 차트) 우(경고 리스트) 2분할 → 하단 최근 공지 3건 리스트
- **UI 요소**: Card(KPI, 아이콘+숫자+전월대비 %), LineChart/BarChart(Recharts), Badge(경고 카운트), List Item
- **사용자 흐름**: 진입 시 React Query가 역할 기반 대시보드 API 호출 → 로딩 Skeleton → 데이터 렌더링 → 기간 필터 변경 시 재요청

## 4.3 직원 관리 (`/employees`, `/employees/:id`)

- **목적**: 임직원 마스터 데이터의 등록/조회/수정/관리
- **주요 기능**: 목록 검색/필터, 등록 모달, 상세 페이지(탭: 기본정보/근태/급여/휴가)
- **화면 구조**: 목록 페이지 — 상단 Toolbar(검색바, 필터 Select, "직원 등록" 버튼) + Table(체크박스, 사번, 이름, 부서, 직급, 상태 Badge, 더보기 메뉴). 상세 페이지 — 좌측 프로필 카드(고정) + 우측 Tab Content.
- **UI 요소**: Table, Pagination, Dropdown Filter, Modal(등록/수정 폼), Badge(재직상태), Tabs, Avatar
- **사용자 흐름**: 목록 → 행 클릭 → 상세 진입 → 탭 전환으로 근태/급여/휴가 이력 조회 → 수정 버튼 → 폼 Modal → 저장 → React Query invalidate → 목록/상세 갱신

## 4.4 급여 관리 (`/payroll`)

- **목적**: 월별 급여 산정, 확정, 지급 처리
- **주요 기능**: 월 선택 → 일괄 급여 테이블 생성, 개별 항목(수당/공제) 수정, 상태 전이(확정/지급), 명세서 다운로드
- **화면 구조**: 상단 월 선택 Dropdown + 상태 요약 Badge(DRAFT N건/CONFIRMED N건) → Table(직원, 기본급, 수당, 공제, 실지급액, 상태, 액션) → 우측 슬라이드 패널(행 클릭 시 상세 수당/공제 breakdown 편집)
- **UI 요소**: Table(우측 정렬 숫자 컬럼), Slide-over Panel, Status Badge(색상: DRAFT 회색/CONFIRMED 파랑/PAID 초록), Confirm Dialog(상태 전이 시)
- **사용자 흐름**: HR_MANAGER가 월 선택 → "급여 일괄 생성" 클릭 → 직원별 기본급 기준 자동 계산 → 개별 보정 필요 시 슬라이드 패널에서 수당/공제 입력 → "확정" 클릭(Confirm Dialog) → 상태 CONFIRMED 전환 → 이후 "지급 처리"로 PAID 전환. EMPLOYEE는 본인 1개월치 급여만 읽기 전용으로 확인.

## 4.5 일정 관리 (`/schedule`)

- **목적**: 개인/부서/전사 일정 공유 및 조율
- **주요 기능**: 캘린더(월/주/일 뷰 토글), 일정 등록(유형/공개범위/참석자), 휴가 신청 연동
- **화면 구조**: 상단 Toolbar(이전/다음, 오늘, 뷰 토글, "일정 추가") → 캘린더 그리드(월 뷰: 7x5 셀, 셀당 일정 최대 3개 + "더보기") → 일정 클릭 시 상세 Popover
- **UI 요소**: Calendar Grid, Event Chip(유형별 색상), Popover(상세), Modal(등록 폼: 제목/유형/기간/공개범위/참석자 Multi-select)
- **사용자 흐름**: "일정 추가" → Modal에서 정보 입력 → 공개범위 선택(전사 공개 시 추가 권한 필요 없음, 등록자 본인 일정이므로) → 저장 → 캘린더에 즉시 반영 → 참석자로 등록된 사용자는 본인 캘린더에도 노출 + 수락/거절 가능

## 4.6 부서 관리 (`/departments`)

- **목적**: 조직 구조 관리
- **주요 기능**: 부서 트리 조회, 부서 추가/이동/삭제, 부서장 지정, 인원 현황
- **화면 구조**: 좌측 Tree View(부서 계층) + 우측 선택된 부서 상세(부서명, 부서장, 소속 인원 Table)
- **UI 요소**: Tree Component(드래그앤드롭으로 상위부서 변경), Select(부서장 지정), Table(소속 인원)
- **사용자 흐름**: 트리에서 부서 선택 → 우측에 상세/인원 표시 → 드래그로 하위 부서 이동 시 Confirm Dialog → 인원 일괄 이동 여부 확인

## 4.7 권한 관리 (`/permissions`)

- **목적**: 역할별 접근 권한 정의
- **주요 기능**: 역할 목록, 모듈x액션 권한 매트릭스 토글, 역할별 소속 인원 확인
- **화면 구조**: 상단 역할 탭(ADMIN/HR_MANAGER/SALES_MANAGER/EMPLOYEE/+커스텀) → 본문 권한 매트릭스 Table(행: 모듈, 열: C/R/U/D/A 체크박스)
- **UI 요소**: Tabs, Checkbox Table, Switch(역할 활성화 여부), Toast(저장 완료 알림)
- **사용자 흐름**: 역할 탭 선택 → 매트릭스 체크박스 토글 → 변경 시 자동 저장(Debounce) 또는 "저장" 버튼 → Toast 확인 → 해당 역할 사용자는 다음 API 호출부터 즉시 반영(JWT 자체에는 권한을 담지 않고 매 요청 시 DB 조회 또는 짧은 TTL 캐시로 반영, [12-jwt-auth-design.md](12-jwt-auth-design.md) 참고)

## 4.8 거래처 관리 (`/partners`)

- **목적**: 고객사/공급사 마스터 관리
- **주요 기능**: 거래처 목록/검색, 등록/수정, 거래처별 거래 이력
- **화면 구조**: 목록(Table: 거래처명, 유형 Badge, 담당자, 등급, 최근거래일) + 상세 Drawer(우측에서 슬라이드, 기본정보 + 거래이력 Tab)
- **UI 요소**: Table, Drawer, Tag(등급 A/B/C 색상), Tabs
- **사용자 흐름**: 목록에서 행 클릭 → Drawer 오픈 → 거래 이력 탭에서 연결된 SalesOrder 목록 확인 → "신규 주문" 버튼으로 영업 주문 생성 화면 진입

## 4.9 제품 관리 (`/products`)

- **목적**: 판매/생산 대상 품목 마스터 관리
- **주요 기능**: 제품 목록/검색/분류 필터, 등록/수정, 단종 처리, 재고/입출고/생산 연계 조회
- **화면 구조**: 목록(Grid 카드 또는 Table 토글) + 상세 페이지(이미지, 기본정보, 탭: 재고현황/입출고이력/생산이력)
- **UI 요소**: Table/Card Toggle, Image Upload, Tabs, Badge(활성/단종)
- **사용자 흐름**: 제품 등록 → SKU 자동 생성 또는 수동 입력 → 안전재고 기준 설정 → 저장 후 재고 모듈에 자동으로 초기 재고 0 레코드 생성

## 4.10 재고 관리 (`/inventory`)

- **목적**: 창고별 실시간 재고 현황 파악
- **주요 기능**: 창고별/제품별 현황 Table, 안전재고 미달 경고, 재고 실사
- **화면 구조**: 상단 창고 선택 탭 + 안전재고 미달 경고 Banner → Table(제품, 현재고, 안전재고, 상태 Badge, 최근 변동일)
- **UI 요소**: Table(낮은 재고 행 빨간 배경 강조), Alert Banner, Button("재고 실사 시작")
- **사용자 흐름**: 재고 실사 시작 → 제품별 실측 수량 입력 폼 → 시스템 수량과 비교하여 차이(+/-) 표시 → 확정 시 자동으로 조정용 StockMovement(ADJUST) 생성

## 4.11 입출고 관리 (`/stock-movements`)

- **목적**: 재고 변동 트랜잭션 등록/이력 조회
- **주요 기능**: 입고/출고 등록, 사유(구매/판매/생산/반품/조정) 분류, 이력 필터링
- **화면 구조**: 상단 Toolbar(기간/제품/창고/유형 필터, "입고 등록"/"출고 등록" 버튼) → Table(일시, 제품, 유형 Badge, 수량(+/- 색상), 사유, 처리자)
- **UI 요소**: Table, Filter Bar, Modal(등록 폼), 수량 +녹색/-빨강 컬러 코딩
- **사용자 흐름**: "출고 등록" → 제품/수량/사유 입력 → 현재고 부족 시 경고 후 강제 진행 여부 확인 → 저장 시 재고 수량 즉시 반영(트랜잭션)

## 4.12 생산 관리 (`/production`)

- **목적**: 생산 오더 진행 상태 추적
- **주요 기능**: 생산 오더 등록, 진행 상태 변경, 지연 오더 하이라이트
- **화면 구조**: Kanban 보드(PLANNED/IN_PROGRESS/DELAYED/COMPLETED 컬럼) 또는 Table 뷰 토글
- **UI 요소**: Kanban Card(제품명, 수량, 마감일 D-day 표시), Badge(DELAYED는 빨간색 강조)
- **사용자 흐름**: 오더 카드를 드래그하여 상태 컬럼 이동 → 완료(COMPLETED) 이동 시 입고 StockMovement 자동 생성 다이얼로그 노출

## 4.13 문서 관리 (`/documents`)

- **목적**: 사내 문서의 중앙 저장소이자 RAG 색인 소스
- **주요 기능**: 업로드(드래그앤드롭), 카테고리/접근범위 설정, 버전 관리, AI 색인 상태 표시
- **화면 구조**: 좌측 카테고리 Tree(정책/계약/보고서/매뉴얼/인사) + 우측 문서 목록(Table: 파일아이콘, 제목, 버전, 업로드자, 색인상태 Badge)
- **UI 요소**: Drag&Drop Upload Zone, Table, Badge("색인 완료"/"색인 중"/"색인 대기"), Version History Modal
- **사용자 흐름**: 파일 드래그 업로드 → 업로드 직후 상태 "색인 대기" → 백그라운드에서 텍스트 추출+Chunking+Embedding 진행 → 완료 시 Badge가 "색인 완료"로 전환되며 AI 챗봇 검색 대상에 자동 포함

## 4.14 공지사항 (`/announcements`)

- **목적**: 전사/부서 공지 전달 및 확인
- **주요 기능**: 작성(Markdown), 고정, 대상 역할 지정, 읽음 확인
- **화면 구조**: 목록(고정 공지 상단 별도 섹션 + 일반 목록) + 상세 페이지(본문 + 읽은 사람 수)
- **UI 요소**: Markdown Editor/Viewer, Pin Icon, Read Count Badge
- **사용자 흐름**: 작성 → 대상 역할 선택(전사 기본) → 게시 → 대상자 로그인 시 Topbar 알림 뱃지 노출 → 클릭 시 상세 진입 및 읽음 처리

## 4.15 통계 분석 (`/statistics`)

- **목적**: 경영/실무 의사결정을 위한 데이터 시각화
- **주요 기능**: 매출/재고/인사 통계 탭, 기간 비교, 차트 다운로드(이미지/CSV)
- **화면 구조**: 상단 탭(매출/재고/인사, 역할에 따라 노출 제한) + 필터(기간) + 차트 Grid(2x2)
- **UI 요소**: Recharts(Line/Bar/Pie), DateRangePicker, Export Button
- **사용자 흐름**: 탭 선택 → 기간 설정 → 차트 렌더링(React Query 캐시) → "CSV 내보내기"로 원본 데이터 다운로드

## 4.16 AI 챗봇 (`/ai-assistant`, 플로팅 위젯)

- **목적**: 자연어로 ERP 데이터/문서에 접근하는 통합 인터페이스
- **주요 기능**: 대화형 질의, 세션 히스토리, 출처 카드(데이터/문서), 추천 질문, 딥링크 이동
- **화면 구조**: 좌측 사이드바(세션 목록, "새 대화") + 중앙 대화창(메시지 버블 + 출처 카드) + 하단 입력창(추천 질문 Chip + Textarea + 전송)
- **UI 요소**: Chat Bubble(사용자/AI 구분 색상), Source Card(테이블 미리보기 또는 문서 인용), Suggested Question Chip, Streaming Typing Indicator
- **사용자 흐름**: 질문 입력 → 전송 → 스트리밍 응답(토큰 단위 표시) → Function Calling 결과는 표 카드로, 문서 검색 결과는 인용 카드로 렌더링 → 카드 내 "자세히 보기" 클릭 시 해당 ERP 화면으로 라우팅

상세 ASCII 와이어프레임은 [06-wireframes.md](06-wireframes.md)에서 7개 핵심 화면(로그인/대시보드/직원관리/급여관리/일정관리/문서관리/AI챗봇)을 1440px 기준으로 제공한다.
