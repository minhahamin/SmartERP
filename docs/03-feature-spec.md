# 3. 기능 명세

표기: **권한**은 [02-users-and-permissions.md](02-users-and-permissions.md)의 매트릭스를 기준으로 한다.

## 3.1 ERP 기능

### 3.1.1 로그인 (Auth)
- 이메일/비밀번호 로그인, JWT Access/Refresh Token 발급
- "로그인 상태 유지" 옵션 (Refresh Token 만료 기간 차등)
- 비밀번호 5회 오류 시 5분 잠금 (Brute-force 방지)
- 최초 로그인 시 비밀번호 변경 강제 플로우 (초대 가입자)
- 상세: [12-jwt-auth-design.md](12-jwt-auth-design.md)

### 3.1.2 대시보드
- 역할별로 다른 위젯 조합 노출 (ADMIN: 전사 KPI, HR: 인사 KPI, SALES: 영업 KPI, EMPLOYEE: 개인 요약)
- 위젯: 매출 추이, 재고 경고, 휴가 현황, 생산 지연 현황, 최근 공지, AI 챗봇 바로가기
- 데이터는 React Query로 캐싱 + 30초 polling 또는 수동 새로고침
- 기간 필터(이번 주/이번 달/이번 분기) 공통 컴포넌트 사용

### 3.1.3 직원 관리 (Employee)
- 목록: 검색(이름/사번/부서), 필터(재직 상태/부서/직급), 페이지네이션
- 등록/수정: 기본정보(이름, 사번, 이메일, 연락처), 인사정보(부서, 직급, 입사일, 재직상태), 권한(Role)
- 상세: 근태 이력, 급여 이력, 휴가 이력을 탭으로 분리하여 조회
- 퇴사 처리: 소프트 삭제(`status = RESIGNED`) + 계정 비활성화 + 인계 메모
- **권한**: ADMIN/HR_MANAGER는 CRUD, EMPLOYEE는 본인 일부 항목(연락처, 주소)만 수정 가능

### 3.1.4 급여 관리 (Payroll)
- 월별 급여 테이블 생성(일괄): 기본급 + 수당(jsonb) - 공제(jsonb) = 실지급액 자동 계산
- 상태: `DRAFT → CONFIRMED → PAID` 상태 전이, CONFIRMED 이후 수정 시 변경 이력 기록
- 급여명세서 PDF 다운로드(개인)
- 일반 직원은 본인 급여 내역만 조회, 금액 항목은 마스킹 없이 본인 데이터 전체 노출(타인 데이터 접근 시 403)
- **권한**: HR_MANAGER만 생성/확정, ADMIN은 전체 열람, EMPLOYEE는 R(own)

### 3.1.5 일정 관리 (Schedule)
- 캘린더 뷰(월/주/일), 일정 유형(회의/휴가/출장/기타) 색상 구분
- 공개 범위: PRIVATE(본인) / DEPARTMENT(부서 공유) / COMPANY(전사 공유)
- 참석자 초대 + 수락/거절 상태 관리
- 휴가 신청과 연동: 휴가 승인이 완료되면 자동으로 일정에 반영

### 3.1.6 부서 관리 (Department)
- 트리 구조(상위부서-하위부서), 부서장 지정
- 부서 이동 시 소속 직원 일괄 이동 처리
- 부서별 인원 수, 평균 근속 등 통계 노출

### 3.1.7 권한 관리 (Permission/RBAC)
- 역할(Role) 목록 관리: 시스템 기본 4종 + 커스텀 역할 추가 가능
- 권한 매트릭스 UI: 모듈(행) x 액션(열) 체크박스 토글, 저장 시 즉시 반영
- 역할별 소속 인원 목록 확인
- **권한**: ADMIN 전용

### 3.1.8 거래처 관리 (Partner)
- 거래처 유형(고객사/공급사/양방), 사업자번호, 담당자(영업사원) 지정
- 거래처별 거래 이력(주문/매출) 탭에서 통합 조회
- 거래처 등급(A/B/C) 태그 관리

### 3.1.9 제품 관리 (Product)
- SKU, 분류, 규격, 단위, 판매가/원가, 안전재고 기준 설정
- 제품 이미지 업로드, 활성/비활성 토글(단종 처리)
- 제품별 재고/입출고/생산 현황을 탭으로 연계 조회

### 3.1.10 재고 관리 (Inventory)
- 창고별 재고 현황 테이블, 안전재고 미달 시 경고 배지(빨간색) 노출
- 재고 실사(Stock Take) 기능: 실측 수량 입력 → 시스템 수량과 차이(편차) 자동 계산 및 조정 입출고 생성
- 재고 이동(창고 간 Transfer) 처리

### 3.1.11 입출고 관리 (Stock Movement)
- 입고(구매/생산 완료/반품) / 출고(판매/생산투입/폐기) 등록
- 입출고 등록 시 재고 수량 트랜잭션 처리(동시성 고려, [07-database-design.md](07-database-design.md) 참고)
- 입출고 이력 조회(기간/제품/창고/유형 필터)

### 3.1.12 생산 관리 (Production) — *확장 모듈*
> 원 요구사항의 ERP 기능 목록에는 명시되지 않았으나, AI 데모 시나리오("생산 지연 중인 작업 목록 보여줘")를 실제로 구현하기 위해 추가한 경량 모듈이다. 실무 제조업 ERP에서 재고/입출고 모듈만으로는 "생산 지연"이라는 개념이 존재하지 않기 때문에, 최소한의 생산 오더 추적 기능을 별도로 둔다.
- 생산 오더 등록: 제품, 계획 수량, 생산 라인, 착수일/완료 예정일
- 상태: `PLANNED → IN_PROGRESS → COMPLETED`, 완료 예정일이 지났는데 미완료면 `DELAYED`로 자동 분류(배치 또는 조회 시점 계산)
- 생산 완료 시 입고(StockMovement IN, ref_type=PRODUCTION) 자동 생성

### 3.1.13 문서 관리 (Document)
- 카테고리(정책/계약/보고서/매뉴얼/인사) 별 업로드, 버전 관리(재업로드 시 버전 +1, 이전 버전 보관)
- 파일 미리보기(PDF), 다운로드, 접근 범위(전사 공개/부서 공개/비공개) 설정
- 업로드 시 자동으로 RAG 파이프라인에 등록(텍스트 추출 → Chunking → Embedding), 상세는 [10-rag-design.md](10-rag-design.md)

### 3.1.14 공지사항 (Announcement)
- 고정(Pin) 공지, 대상 역할 필터(전사/특정 역할만), 읽음 확인(Read Receipt)
- 작성 시 Markdown 에디터 지원, 첨부파일 가능

### 3.1.15 통계 분석 (Statistics)
- 매출 추이(월별/제품별/거래처별), 재고 회전율, 근태 통계(지각/결근율), 인건비 추이
- Recharts 기반 시각화(Line/Bar/Pie), 기간 비교(전월 대비, 전년 대비) 기능
- 역할별 통계 범위 제한(영업은 매출만, HR은 인사만, ADMIN은 전체)

## 3.2 AI 기능

AI 기능은 단일 진입점인 **AI 업무 도우미(챗봇)** 를 통해 제공되며, 내부적으로 4가지 처리 경로로 라우팅된다.

### 3.2.1 AI 업무 도우미 (통합 챗봇 UI)
- 화면 우측 하단 플로팅 버튼 또는 전용 페이지(`/ai-assistant`)로 접근
- 대화 세션(ChatSession) 단위로 히스토리 저장, 좌측에 과거 대화 목록 노출(ChatGPT 스타일)
- 응답에 출처 표시: ERP 데이터 조회 결과는 "조회 조건" 카드, 문서 검색 결과는 "참고 문서 N개" 인용 카드로 구분 노출

### 3.2.2 자연어 질의 → ERP 데이터 조회
- 사용자의 자연어 질문을 LLM이 분석하여 사전에 정의된 Function(Tool) 중 적합한 것을 선택·호출
- 예: "현재 재고가 100개 이하인 품목 알려줘" → `getLowStockProducts({ threshold: 100 })` 호출 → DB 조회 → 결과를 표 형태로 렌더링 + 자연어 요약 문장 생성
- 모든 Function 호출에는 호출자의 `userId`, `role`, `departmentId`가 서버 측에서 강제 주입되어 권한 범위를 벗어난 조회가 불가능
- 상세: [09-ai-chatbot-design.md](09-ai-chatbot-design.md)

### 3.2.3 사내 문서 검색 (시맨틱 검색)
- 키워드 매칭이 아닌 의미 기반 검색: "신입사원 연차 규정"이라는 질문이 문서 내 "입사 1년 미만 직원의 연차 산정 기준"과 매칭되도록 임베딩 유사도 검색
- pgvector를 이용한 Cosine Similarity Top-K 검색 → 관련 Chunk를 LLM 컨텍스트에 주입하여 답변 생성(RAG)
- 상세: [10-rag-design.md](10-rag-design.md), [11-pgvector-design.md](11-pgvector-design.md)

### 3.2.4 ERP 데이터 조회 (구조화 데이터 질의)
- 3.2.2와 동일한 Function Calling 경로를 사용하되, "조회"뿐 아니라 "요약/집계" 질의("이번 달 매출 요약해줘")는 집계 함수(`getSalesSummary`)를 호출
- 응답에는 항상 원본 데이터로 드릴다운할 수 있는 "상세 보기" 링크 포함(해당 ERP 화면으로 딥링크)

### 3.2.5 문서 요약
- 문서 관리에서 업로드된 PDF/Word 문서를 업로드 직후 비동기로 요약(3~5줄 핵심 요약 + 키워드 태그) 생성
- 사용자가 챗봇에서 "OO 계약서 요약해줘"라고 요청하면 해당 문서의 Chunk들을 모아 LLM이 재요약(Map-Reduce 요약)

### 3.2.6 FAQ 자동 생성
- ChatMessage 로그를 주기적(예: 매일 새벽 배치)으로 분석하여 의미적으로 유사한 질문을 클러스터링
- 동일 의도의 질문이 임계치(예: 5회) 이상 누적되면 FAQ 후보로 추출 → 관리자가 검수 후 게시(Human-in-the-loop)
- 게시된 FAQ는 이후 동일 질문 발생 시 Function Calling/RAG 없이 캐시된 답변을 우선 매칭하여 응답 속도와 비용을 절감

## 3.3 기능-모듈 의존 관계 요약

```
로그인 ──┬─→ 모든 화면 (Auth Guard)
         └─→ AI 챗봇 (JWT 클레임 기반 권한 주입)

직원관리 ──→ 급여관리 / 근태(직원관리 하위) / 일정관리 / 권한관리
부서관리 ──→ 직원관리 / 통계분석

거래처관리 ──→ 제품관리 ──→ 재고관리 ──→ 입출고관리 ──→ 생산관리
                                              └─→ 통계분석

문서관리 ──→ RAG 파이프라인 ──→ AI 문서검색 / 문서요약 / FAQ자동생성
ERP 전 모듈 ──→ AI ERP데이터조회 (Function Calling)
```
